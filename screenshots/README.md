## Screenshot Guidelines

### Technical Requirements

- **Format**: PNG (preferred) or JPG
- **Resolution**: Minimum 1920x1080 pixels
- **File Size**: Optimize for web (aim for < 500KB per image)
- **Naming**: Use exact filenames as specified in the manual

### Content Guidelines

1. **Clarity**: Ensure screenshots are clear and well-lit
2. **Relevance**: Capture only the relevant interface elements
3. **Annotations**: Add arrows, boxes, or text annotations to highlight important elements if needed
4. **Privacy**: Blur or redact any sensitive information (personal data, real user names, etc.)
5. **Consistency**: Use consistent browser/theme settings across all screenshots

### Capturing Screenshots

#### For Admin Dashboard:
1. Log in with test credentials: `akwatuha` / `reset123`
2. Navigate to the relevant page
3. Use browser developer tools (F12) to ensure clean view
4. Capture full page or specific sections as needed
5. Use browser extensions like "Full Page Screen Capture" for long pages

#### For Public Dashboard:
1. Open in incognito/private browsing mode
2. Navigate to the relevant page
3. Capture the interface
4. Ensure no personal data is visible

### Tools for Screenshot Capture

- **Windows**: Snipping Tool, Windows + Shift + S, or ShareX
- **Mac**: Cmd + Shift + 4, or Skitch
- **Linux**: Screenshot tool, or Flameshot
- **Browser Extensions**: Full Page Screen Capture, Awesome Screenshot

### Annotation Tools

- **Windows**: Snipping Tool, Paint, or Greenshot
- **Mac**: Preview, Skitch, or Annotate
- **Online**: Annotely, Snagit, or Canva

When the system interface changes:

2. Capture new screenshots following guidelines

4. Update manual if interface changes significantly
